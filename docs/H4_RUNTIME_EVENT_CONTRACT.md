# Purpose

Define the official Typed Event Contract for Phantom Runtime Lite.

This contract defines the only public Runtime Event API exposed by the Cloud Run Runtime.

The Cloud Run Runtime remains the sole execution engine.

All downstream components consume Typed Events only.

Supported downstream components include:

* Verification Runtime
* Trust Runtime
* Event Aggregator
* Dashboard
* FastAPI

# Design Principles

## P1. Runtime Independence

The Cloud Run Runtime publishes Typed Events only.

It must not depend on downstream runtimes.

---

## P2. Event-Driven Extension

All runtime extensions consume Typed Events.

Direct access to Runtime internal state is prohibited.

---

## P3. Immutable Events

Published events are immutable.

Subscribers must never modify an event.

---

## P4. Ordered Event Stream

Events preserve publishing order within the same session.

Ordering is guaranteed by:

* sequence
* timestamp

---

## P5. Provider Independence

The event schema is provider-independent.

Provider-specific fields are prohibited except informational metadata.

# Runtime Event Envelope

Every Runtime Event shares the following envelope.

```text
RuntimeEvent

schema_version: str

event_id: str

timestamp: datetime

session_id: str

sequence: int

type: EventType

payload: EventPayload

metadata: EventMetadata (optional)
```

---

# Runtime Event Metadata

metadata is a reserved area for Runtime Event supplementary information.

metadata is independent of payload. The meaning of payload is not changed by metadata.

metadata is optional. Its absence never raises and downstream Runtimes treat it as empty.

## Purpose

metadata formally defines the Conversation Traceability responsibility of the Runtime Event: which Runtime Conversation line, spoken by which speaker, produced this event.

metadata is also the common extension point for future Runtime Event supplementary fields, so that future additions do not require new top-level envelope fields.

## Fields currently in use

| Field | Type | Description |
|--------|------|-------------|
| conversation_line | int? | Runtime Conversation の発話番号 |
| speaker | string? | YOU / AGT |
| transcript | string? | Runtime Conversation の発話内容 |

## Reserved fields (not yet in use)

The following fields are reserved for future use. This contract revision does not read, write, or generate any of them:

* trace_id
* runtime_version
* websocket_id
* provider
* provider_model
* latency
* device
* microphone
* observation_id

## Example

```json
{
  "version": 1,
  "type": "transcript",
  "timestamp": "...",

  "payload": {
      ...
  },

  "metadata": {
      "conversation_line": 31,
      "speaker": "YOU",
      "transcript": "現在、利用人数はどのくらいを想定されていますか？"
  }
}
```

## Compatibility

metadata follows the same Backward Compatibility rules as payload (see "Backward Compatibility" below): it is an optional addition within schema_version 1.0 and does not require a new major schema version.

---

# Schema Versioning

The Runtime Event Contract is versioned.

Rules:

* schema_version identifies the event schema.
* Consumers shall reject unsupported schema versions.
* Minor schema versions must remain backward compatible.
* Breaking changes require a new major schema version.

Current schema version:

```text
1.0
```

# Event Payloads

## transcript

Generated by Whisper.

```text
text: str

language: str

confidence: float

is_final: bool
```

---

## reply

Generated by the selected provider.

```text
provider: str

model: str

text: str

finish_reason: str
```

### Provider Rules

provider is informational metadata only.

Consumers must treat provider as an arbitrary string.

Consumers must not enumerate supported providers.

---

## analysis

Optional runtime analysis.

```text
intent: str

summary: str

metadata: dict
```

### metadata Rules

metadata must be JSON serializable.

Allowed value types

* string
* number
* boolean
* null
* object
* array

Custom runtime objects are prohibited.

metadata may contain only provider-independent information.

---

## latency

Performance metrics.

```text
stt_ms: int

routing_ms: int

provider_ms: int

total_ms: int
```

---

## status

Runtime lifecycle.

```text
state: RuntimeState

message: str
```

Supported states

```text
STARTING

READY

PROCESSING

IDLE

STOPPED
```

---

## error

Runtime error information.

```text
code: str

message: str

recoverable: bool
```

# Event Ordering

Within the same session:

* sequence is monotonically increasing.
* timestamp must not move backwards.
* Missing sequence numbers are allowed.
* Reverse ordering is prohibited.

---

# Thread Safety

RuntimeEvent objects are immutable.

Only publishers create RuntimeEvents.

Subscribers operate in read-only mode.

---

# Subscriber Contract

Every subscriber exposes:

```text
handle(RuntimeEvent)
```

Requirements

* Read-only
* No mutation
* No return value

---

# Delivery Semantics

Typed Events use asynchronous fire-and-forget delivery.

Publishers do not wait for subscriber acknowledgements.

Retry policies, persistence, message durability and dead-letter queues are outside the scope of this contract.

---

# Failure Isolation

Subscriber failures must never interrupt Runtime execution.

Publisher continues delivering events to remaining subscribers.

Verification Runtime failure must not stop Trust Runtime.

Trust Runtime failure must not stop Dashboard.

Dashboard failure must not affect Runtime.

FastAPI failure must not affect Runtime.

---

# Serialization

All RuntimeEvents must be JSON serializable.

The same schema is shared by:

* Cloud Run Runtime
* Verification Runtime
* Trust Runtime
* Event Aggregator
* Dashboard
* FastAPI

---

# Backward Compatibility

Within the same major schema version:

* Consumers must ignore unknown fields.
* Publishers must never remove existing fields.
* Publishers may add optional fields only.
* Existing field semantics must not change.

Breaking changes require a new major schema version.

---

# Compatibility

The contract is provider-independent.

Supported providers include:

* OpenAI
* Gemini

Future providers must comply with this contract without requiring schema changes.
