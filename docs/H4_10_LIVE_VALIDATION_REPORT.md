## 2. Runtime Event Generation

The real, unmodified `src/phantom_runtime.py` was imported as a library
(never executed as `__main__`, so no audio-capture/keyboard threads were
started) and driven directly through its own real functions using genuine
OpenAI traffic:

- Real speech audio (synthesized via macOS `say`, 2.49s) →
  `transcribe()` → real `client.audio.transcriptions.create()` (Whisper)
- `generate_agent_reply()` → real streaming Chat Completions call
- `generate_meeting_analysis()` → real non-streaming Chat Completions call
- `_set_state()`, `show_latency()` → real, deterministic Runtime functions

All five Contract event types were confirmed emitted by the real,
unmodified `_emit_event()`, captured over a real pipe (`PHANTOM_EVENT_FD`,
the same H3 transport mechanism `runtime.cloud_run_shell` wires in
production):

| type | payload (as emitted) |
|---|---|
| transcript | `text="Hello, when can you start the new position?"` (real Whisper output), `lang="english"`, `speaker="recruiter"` |
| status | `state="recruiter_speaking"`, `previous="idle"` |
| reply | `text="I can start as soon as needed, but I'd prefer to discuss any notice period if applicable."` (real Chat Completions output) |
| latency | `stt_ms=2089.26`, `gpt_ms=1383.98`, `total_ms=3473.24` (real measured durations) |
| analysis | full real structured meeting analysis text (Japanese, generated by Chat Completions) |

Result: **PASS** — all 5 required Runtime Event types generated from real
API-derived data, no mocking.

*Note (transparency, not a validation failure):* driving
`generate_meeting_analysis()` live had the side effect of writing one
real entry each into `src/memory/{decision,question_cluster,question,
rolling_summary,subject_registry}_memory.json` (pre-existing, empty,
untracked memory-store files). This was identified immediately after the
run and reverted — all five files were restored to their original empty
`[]` state before the next validation step began. `git status` was
confirmed unchanged before and after every step in this validation.
